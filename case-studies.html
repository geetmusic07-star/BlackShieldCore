Supply Chain Compromise via Dependency Confusion in a CI/CD Pipeline

Reconstruction of an internal package substitution scenario: a private npm package name was registered publicly with a higher version number. The build pipeline resolved to the malicious public version, executing attacker-controlled code during the CI build.

Scenario · Apr 2026·45 min read
Attack Vector: Dependency Confusion (npm)
Package manager resolved public registry over private registry for the same package name. Attacker-controlled version executed preinstall scripts with network access during build.
Dwell time: 14 days in CI environment before anomalous outbound connections triggered alert. Three production builds executed the malicious package without detection.

Ransomware Deployment via Active Directory Lateral Movement

Analysis of a ransomware scenario where initial access via a phishing email escalated to domain compromise in under 4 hours. The attack used Kerberoasting to extract service account credentials, followed by GPO modification to deploy the ransomware payload to domain-joined workstations.

Scenario · Mar 2026·38 min read
Initial Access → Domain Admin: 3h 47m
Service account passwords had not been rotated in 18+ months. Kerberoasted hash cracked in under 2 minutes with a 10-word GPU rig on a medium-complexity dictionary.
GPO-based deployment bypassed endpoint protection on 87% of domain-joined machines. Detection triggered only after file system encryption began.

LLM-Assisted Application: Indirect Prompt Injection Enabling Data Exfiltration

Demonstration scenario where an LLM-powered document assistant was manipulated via malicious content embedded in uploaded documents. The injected instructions redirected the assistant to summarize and transmit session data to an attacker-controlled URL.

Scenario · Feb 2026·32 min read
Attack Type: Indirect Prompt Injection
Malicious instructions embedded in a PDF document were processed as trusted content. No input sanitization was applied to document text before passing it to the model context.
The model completed the exfiltration task without any safety system triggering, as the malicious instruction was framed as a "document formatting requirement."

Insider Exfiltration: DLP Bypass via Legitimate Cloud Storage Channels

Analysis of a data exfiltration scenario by a privileged user who bypassed DLP controls by staging sensitive data through corporate-approved cloud storage with minor obfuscation techniques. The scenario highlights gaps in content inspection for authorized storage platforms.

Scenario · Jan 2026·28 min read
Bypass Method: Legitimate Cloud Sync
DLP policies blocked direct upload to personal storage but allowed sync via corporate-approved tools. Rename + compress obfuscation was sufficient to avoid content-based detection.
UEBA flagged the behavior 6 days after exfiltration began due to unusual access volume patterns. Earlier detection would have required deeper file content inspection.

Cloud Privilege Escalation via IAM Misconfiguration and Metadata Service Abuse

Reconstruction of an AWS environment compromise starting from an overpermissioned IAM role attached to a publicly accessible EC2 instance. SSRF access to the metadata service yielded credentials that permitted S3 bucket enumeration, data access, and further privilege escalation.

Scenario · Dec 2025·35 min read
Entry Point: SSRF → EC2 Metadata
IMDSv1 was enabled with no hop count restriction. SSRF via a web application parameter retrieved role credentials within a single HTTP request.
The IAM role permitted sts:AssumeRole on a second role with S3 full access. No CloudTrail alerting was configured for cross-role assumption from EC2.
